MegaBrainMarket 附有许多**磁贴式**指标、关联和信号面板 — 紧凑的面，每个显示一两个读数，设计为与深度分析面板组合使用。本页是集合参考，使每个磁贴可查找，而无需为每个面板写一页。

当磁贴为空时，请检查 [数据来源](/zh/data-sources) 和 [健康端点](/zh/health-endpoints) 了解上游状态。

## 基础设施支撑的端点

紧凑的基础设施磁贴使用生成的 InfrastructureService 客户端。`internet-disruptions`、DDoS 摘要、流量异常、`service-status` 以及服务端计算的时序异常会先读取 bootstrap 数据（可用时），再在 bootstrap 数据缺失、被过滤或过期时调用 REST RPC。IP 地理定位、反向地理编码、bootstrap 缓存读取和基线写入等工具路由属于数据面路由，而非独立磁贴。

| 方法 | 路由 | 所供数据 |
|--------|-------|---------------|
| `GET` | `/api/infrastructure/v1/list-internet-outages` | 来自 Cloudflare Radar 的互联网中断地图/磁贴数据；`fetchInternetOutages()` 先用 bootstrap `outages`，再用此 RPC。 |
| `GET` | `/api/infrastructure/v1/list-service-statuses` | 服务状态磁贴；`fetchServiceStatuses()` 先用 bootstrap `serviceStatuses`，再用此 RPC（可选状态过滤）。 |
| `GET` | `/api/infrastructure/v1/get-cable-health` | 电缆图层与基础设施级联上下文；请参见 [Infrastructure Cascade Analysis](/zh/infrastructure-cascade) 了解电缆专属工作流。 |
| `GET` | `/api/infrastructure/v1/get-temporal-baseline` | 对军用航班、船舶、抗议、AIS 间隙和卫星火灾等指标的客户端时序异常检查。 |
| `GET` | `/api/infrastructure/v1/list-temporal-anomalies` | 新闻与卫星火灾的服务端计算时序异常；先消费 bootstrap `temporalAnomalies`，随后 `fetchLiveAnomalies()` 可调用此路由。 |
| `GET` | `/api/infrastructure/v1/list-internet-ddos-attacks` | 来自 Cloudflare Radar 的 DDoS 攻击摘要；`fetchDdosAttacks()` 先用 bootstrap `ddosAttacks`，再用此 RPC。 |
| `GET` | `/api/infrastructure/v1/list-internet-traffic-anomalies` | 来自 Cloudflare Radar 的流量异常事件；未过滤视图使用 bootstrap `trafficAnomalies`，而按国家过滤的读取调用此 RPC。 |
| `GET` | `/api/infrastructure/v1/get-ip-geo` | 用于基础设施感知体验的调用方 IP 地理定位工具；无面板直接从其水合。 |
| `GET` | `/api/infrastructure/v1/reverse-geocode` | 面向 API 使用者与 v1 调用方的坐标到地点查找工具；部分 UI 路径仍使用遗留的轻量 `/api/reverse-geocode` 助手。 |
| `GET` | `/api/infrastructure/v1/get-bootstrap-data` | 针对所选 bootstrap/cache 键的批量缓存读取 RPC；补充顶层 bootstrap 水合路径。 |
| `POST` | `/api/infrastructure/v1/record-baseline-snapshot` | 时序异常追踪使用的后台基线更新；它写入观测值，自身不渲染可见磁贴。 |

## 共享行为

- 以上所有面板均**免费**，除非另有说明 — 在 `src/config/panels.ts` 中无 `premium: 'locked'`。
- 紧凑磁贴跨多个变体注册；按变体的注册决定磁贴是否默认启用。确认可用性时请直接检查 `src/config/panels.ts`。
- 大多数磁贴式面板从 bootstrap 载荷（`getHydratedData(...)`）水合，并回退到新的 RPC 调用，使初始绘制即时完成。
