**My Monitors** 面板（内部 id `monitors`）是仪表盘的个人关键词提醒面。输入逗号分隔的关键词，应用中每个新闻源里任何匹配的文章都会以你监控器分配的颜色高亮 — 包括聚类内的文章。监控器通过 `localStorage` 跨会话持久化。

关于监控器行为的更详细描述见 [功能 → 自定义监控器](/zh/features#custom-monitors)；本页是简短的面板参考。

## 面板显示内容

- **输入行** — 一个用于逗号分隔关键词的文本框和一个添加按钮。在输入框中按 Enter 也会添加监控器。
- **监控器行** — 每个注册的监控器显示其关键词、分配的颜色标签和一个移除交互。
- **调色板** — 来自 `@/config` 中的 `MONITOR_COLORS`；每个新监控器自动分配颜色，使各行保持视觉上的区分。

面板外的行为：

- **每个新闻源**（实时新闻、区域源等）中的匹配文章以监控器的颜色作为强调色渲染。
- 在新闻**聚类**内，子文章继承监控器颜色，因此即使在聚类折叠时监控器匹配也可见。

面板 id 为 `monitors`；规范组件为 `src/components/MonitorPanel.ts`。标题来自 i18n（`panels.monitors`）；配置名："My Monitors"。

## 如何访问

- **Cmd+K**：输入 *monitor* 或 *keyword*。
- **按变体的可用性**：在 **full/geopolitical**（`priority: 2`）、**tech**（`priority: 2`）、**finance**（`priority: 2`）和 **commodity**（`priority: 2`）变体中注册并默认启用。在 happy 变体中不存在。来源：`src/config/panels.ts` 中 `FULL_PANELS`（`:62`）、`TECH_PANELS`（`:282`）、`FINANCE_PANELS`（`:473`）、`COMMODITY_PANELS`（`:775`）的 `monitors` 条目。

## 数据来源

无 — 监控器完全在客户端运行。关键词列表持久化到浏览器 `localStorage`，因此在无账户的情况下也能在重新加载后保留。匹配是对传入新闻条目在应用内应用的子串检查。

## 刷新频率

实时、客户端：匹配在新闻条目到达时应用；无轮询或服务端频率。

## 层级与门控

**免费。** 任何变体注册中均无 `premium` 标志。
