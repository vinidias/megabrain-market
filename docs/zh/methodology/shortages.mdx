## 我们追踪什么

四类产品，两个严重性层级，全球覆盖：

| 产品      | `watch` 层级 | `confirmed` 层级 |
|-----------|--------------|------------------|
| 喷气燃料  | ✓ | ✓ |
| 汽油      | ✓ | ✓ |
| 柴油      | ✓ | ✓ |
| 取暖油    | ✓ | ✓（仅冬季） |

仅价格飙升不是短缺信号。我们以物理供应约束为键：航班取消、加油站关闭、正式配给、进口削减。

## 全自动化 — 无人工审核队列

MegaBrainMarket 完全自动运行每个分类器。短缺层级也不例外。质量来自**分层证据阈值 + LLM 二次合理性检查 + 过时行自动衰减**，而非来自人工审核员。

### watch 层级（自动化）

一个可信来源：

- 监管机构公告（EPRA、NMDPRA、OGRA、CAA UK、DGCA），**或**
- 航空公司 / 机场运营公告，**或**
- 带有 ≥ 1 个佐证观测的国家级通讯社报道

### confirmed 层级（自动化，更严格的标准）

`watch → confirmed` 的提升需要：

- **两个不同的新闻机构 + 一个监管机构**，或
- **三个不同的新闻机构**，或
- **一个监管机构 + 一个直接运营影响信号**（航空公司信息流中的航班取消、正式配给公告、加油站关闭列表）

并且 — LLM 二次检查必须同意该提升。如果 LLM 不同意，该行保持 `watch`，直到有更多来源超过阈值。

### 自动衰减

- `confirmed` 在 7 天内没有新的佐证信号 → 自动降级为 `watch`
- `watch` 在 14 天内没有新信号 → 自动移除

当上线后的分类器发布时，过时的短缺永远不会静默持续——每次降级都会向计划的公开修订日志写入一条记录。请参阅 [`/corrections`](/zh/corrections) 了解设计好的审计界面形状与当前状态。目前上述降级阈值就是契约；结构化审计追踪随分类器一起上线。

## 证据透明度

每个公开短缺行内联暴露其 `evidenceSources[]` — 您可以阅读分类器使用的来源。这就是让 MegaBrainMarket 的短缺登记表在没有编辑队列的情况下也能站得住脚的原因。

对于任何 `confirmed` 行，面板展示：

- 新闻机构 / 监管机构名称
- 来源日期
- 分类器版本 + 置信度

代理（MCP 客户端）通过 `ListFuelShortages` RPC 接收相同的结构化证据——响应中的每个 `FuelShortageEntry` 都包含完整的 `evidence.evidenceSources[]` 数组，与 UI 抽屉渲染的内容一致。

## 根因归因

每个短缺行都带有一个原因链，而不仅仅是严重性：

- `chokepoint` — 追溯到咽喉要道状态变化的物理供应约束
- `pipeline_disruption` — 登记表中特定管道离线
- `sanction` — 新的制裁机构名单触发进口削减
- `upstream_refinery` — 炼油厂检修或停电
- `logistics` — 港口 / 铁路 / 卡车瓶颈
- `policy` — 政府故意限制

## 紧急覆盖（计划中）

一个 `energy_asset_overrides` 持久化层是设计好的紧急界面，用于读者标记分类明显错误的罕见情况。写入将是仅管理员、非关键路径——默认流程保持完全自动化，且每次覆盖都会向修订日志发出一个 `override` 触发条目。

**状态（v1 上线）：** 覆盖持久化层尚未实现。读者标记的更正目前通过 [GitHub issues](https://github.com/vinidias/megabrain-market/issues) 人工处理；一旦该层随分类器上线，将通过自动化的覆盖路径流转。请参阅 [`/corrections`](/zh/corrections) 了解计划中覆盖条目的形状。

## 已知限制

- 非英语监管机构信息流会有一定延迟；我们在逐步添加它们。
- 在政治噪声较大的国家，"watch" 层级可能会频繁波动——这是有意的；如果读者想要更严格的视图，可以过滤为仅 `confirmed`。
- 取暖油短缺是季节性的，且在通讯社报道中代表性不足；冬季月份信号更强。
